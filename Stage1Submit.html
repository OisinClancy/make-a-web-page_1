Lesson 1

World Wide Web - Consists of a massive system of interconnected computers that communicate with each other through use of HTML Documents.

HTML - Hyper Text Markup Language

Tags - Different elements are found inside brackets such as < > or closing brackets < / >

Elements - These are the properties that are found inside tags such as 'b' for bold and 'em' for italic

Attribute - These are attached to elements to help such as 'src' for an image or 'href' for a link

Inline/Block - Two different types of elements. Block elements form 'invisivble boxes' around the content they surround.

Different Types of Code

  • HTML - Structure

DOM - A general convention for the format of how to write code regardless of language. The nodes of every document are organized in a tree structure.

When looking at html pages it is useful to imagine everything as boxes within boxes. All the elements are boxes that may have have smaller boxes within them or be contained within lager ones. The use of a < div > tag allows you to create these boxes.

Lesson 3

Repetition - It is very important to try and avoid repetition in programming by unecessarily rewriting the same code over and over. It is better to try and group similar elements under the same class and apply styles . This means less code needs to be written and makes it easier if there are mistakes to be corrected or changes to be made.

Selector - This tells the browser what element you want the rules to apply to. In between the curly braces { } is where the rule is placed. There should be a 'property' and a 'value' such as font-family: futura

Comments - These are lines of text that the computer ignores. They are just there so that programmers can leave notes for other people (or themselves) to read.

  • In HTML, code comments begin with < !-- and end with -- >. Everything in between is ignored.
  • In CSS, code comments begin with /* and end with */.

Box Sizing - HTML elements are boxes and each box has 4 components. These are the content, padding, border and margin.Because there are so many components to each box, it can often be hard to get the size of a box just right. There are two techniques you can use to help deal with sizing issues.

  • Set sizes in terms of percentages rather than pixels.
  • Set the box-sizing attribute to border-box for every element.

Box Positioning - Divs are block elements (as opposed to inline), so by default they take up the entire width of a page. Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
